Documentación del API WayFinder

Bienvenidos a nuestra API del transporte público de Santo Domingo, más específicamente de los carros públicos. Utilizando Node.js y Express, nos conectamos a una base de datos SQL Server con alta disponibilidad en Azure para ofrecerte información en tiempo real sobre las rutas de transporte. Nuestros datos GTFS están disponibles para potenciar tus aplicaciones de mapeo, planificación de transporte y análisis de tráfico. ¡Comienza a explorar y descubre lo que puedes crear!

Cómo empezar a probar

Para utilizar nuestra API puedes utilizar:

  • Cliente HTTP: Postman, Thunder Client, o tu cliente de preferencia.
  • Node.js: (Opcional) Si quieres interactuar con nuestra API desde tu código.

Requisitos

  • Cliente HTTP (Postman, Thunder Client, etc.)
  • Node.js (para desarrollo local)

Pasos

  1. Descarga e instala un cliente HTTP.
  2. (Opcional) Si planeas desarrollar localmente, asegúrate de tener Node.js instalado.
  3. ¡ A continuación sigue los pasos que se muestran más adelante y estarás listo para desarrloar con nuestra API !

Recuerda, si tienes alguna duda, estamos aquí para ayudarte.

Endpoint:


fetch('https://gtfs-restapi-production.up.railway.app/routes')
.then(response => response.json())
.then(data => console.log(data));

/routes

Este endpoint devuelve todas las rutas de transporte disponibles en Santo Domingo.

Metodo: GET

Respuesta: Un arreglo de objetos, donde cada objeto representa una ruta de transporte y contiene detalles como el ID de la ruta, el nombre de la ruta, el tipo de vehículo y más.

Estructura de la respuesta:


{"route_id":105,"agency_id":1,"route_short_name":"C-99 B",
"route_long_name":"Carretera Mella Esq. Av. Charles deGaulle (Km. 9)
 a H7C/Miguel Diaz Esq. C/Francisco de Porras.","route_type":5}

Respuestas de la API

Nuestra API responde con códigos de estado HTTP estándar para indicar el éxito o fracaso de una solicitud. Las respuestas exitosas devolverán un cuerpo de respuesta en formato JSON, que sigue la estructura de un documento GTFS estándar.

Por ejemplo, una solicitud GET a cualquier endpoint puede devolver una respuesta exitosa (HTTP 200) con un cuerpo de respuesta que contiene un arreglo de objetos JSON.

Casos de uso

Nuestra API puede ser de gran utilidad para desarrolladores y empresas que trabajen en el desarrollo de aplicaciones de transporte, análisis de rutas, o cualquier proyecto que requiera de datos precisos y actualizados del sistema de transporte público de Santo Domingo. Tal y como mostramos en los siguientes puntos.

  1. Optimización de Rutas Inteligentes: Utiliza nuestros datos en tiempo real para desarrollar algoritmos de optimización de rutas avanzados. Ayuda a las flotas de carros públicos a planificar rutas eficientes en función del tráfico actual, reduciendo los tiempos de espera y mejorando la experiencia de los pasajeros.
  2. Análisis Predictivo del Flujo de Pasajeros: Aprovecha los datos históricos y en tiempo real para predecir la demanda de pasajeros en diferentes rutas y horarios. Esto permite a las empresas de transporte ajustar sus operaciones según las tendencias de demanda, asignando recursos de manera óptima y mejorando la satisfacción del usuario.
  3. Plataforma de Información Multimodal: Crea una aplicación que integre datos de diferentes modos de transporte, como carros públicos, metro, autobuses y más. Ofrece a los usuarios opciones de viaje completas y coherentes, facilitando la planificación de rutas y fomentando el uso de transportes públicos en la ciudad.
Estos casos de uso pueden transformar la forma en que se aborda el transporte público en Santo Domingo y brindar soluciones innovadoras a los desafíos actuales del sistema de transporte.
