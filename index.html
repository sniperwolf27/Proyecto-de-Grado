Documentación de API para mi servicio GTFS

Bienvenidos a la API del transporte público de Santo Domingo. Utilizando Node.js y Express, nos conectamos a una base de datos MS SQL con alta disponibilidad en Azure para ofrecerte datos en tiempo real sobre rutas de transporte. Nuestros datos GTFS están aquí para potenciar tus aplicaciones de mapeo, planificación de transporte y análisis de tráfico. Empieza a explorar y ve lo que puedes crear!

Cómo empezar

Para utilizar nuestra API necesitas:

  • Cliente HTTP: Postman, Thunder Client, o tu cliente de preferencia.
  • Node.js: (Opcional) Si quieres interactuar con nuestra API desde tu código.

Requisitos

  • Cliente HTTP (Postman, Thunder Client, etc.)
  • Node.js (para desarrollo local)

Pasos

  1. Descarga e instala un cliente HTTP.
  2. (Opcional) Si planeas desarrollar localmente, asegúrate de tener Node.js instalado.
  3. ¡Estás listo para explorar nuestra API!

Recuerda, si tienes alguna duda, estamos aquí para ayudarte.

Endpoint:


fetch('https://gtfs-restapi-production.up.railway.app/routes')
.then(response => response.json())
.then(data => console.log(data));

/routes

Este endpoint devuelve todas las rutas de transporte disponibles en Santo Domingo. Los datos están en formato GTFS y se actualizan regularmente para reflejar los cambios en las rutas de transporte.

Metodo: GET

Parámetros: Ninguno

Respuesta: Un arreglo de objetos, donde cada objeto representa una ruta de transporte y contiene detalles como el ID de la ruta, el nombre de la ruta, el tipo de vehículo y más.

Respuestas de la API

Nuestra API responde con códigos de estado HTTP estándar para indicar el éxito o fracaso de una solicitud. Las respuestas exitosas devolverán un cuerpo de respuesta en formato JSON, que sigue la estructura de un documento GTFS estándar.

Por ejemplo, una solicitud GET a cualquier endpoint puede devolver una respuesta exitosa (HTTP 200) con un cuerpo de respuesta que contiene un arreglo de objetos JSON.

Casos de uso

Nuestra API puede ser de gran utilidad para desarrolladores y empresas que trabajen en el desarrollo de aplicaciones de transporte, análisis de rutas, o cualquier proyecto que requiera de datos precisos y actualizados del sistema de transporte público de Santo Domingo.
